ESP32-WROOM-32D · MicroPython v1.25.0
Antes de comenzar: Usa Google Chrome o Edge. Conecta el ESP32 por USB. La conexion al chip (pasos 1 y 2) usa auto-reset por DTR/RTS. Si la conexion falla, manten presionado BOOT al seleccionar el puerto y sueltalo cuando el proceso inicie. Los pasos 3 en adelante no requieren BOOT. Importante: Los pasos 3 a 6 reinician el ESP32 al terminar. Si el dispositivo ya estaba autenticado (enviando datos al servidor), la sesion anterior en Redis bloqueara la re-autenticacion (error 409). En ese caso, ejecuta el flush de Redis antes o despues de la operacion; el monitor serial (paso 5) detecta el error automaticamente y muestra el comando exacto para resolverlo.
1

Diagnostico de Chip

Detecta el modelo, revision, MAC y capacidad de flash del ESP32 conectado. Si la conexion falla, manten BOOT al seleccionar el puerto.

2

Flash MicroPython

Graba el firmware MicroPython v1.25.0 en la flash del ESP32. La opcion erase-first borra toda la flash antes de escribir, garantizando una instalacion limpia. Si la conexion falla, manten BOOT al seleccionar el puerto.

Tu navegador no soporta Web Serial. Usa Chrome o Edge. No se puede acceder al puerto serial.
3

Subir Archivos al ESP32

Transfiere los 14 modulos .py del firmware IoT al sistema de archivos del ESP32 via el protocolo Raw REPL. Si tienes un config.json cargado, tambien se sube automaticamente. El proceso toma entre 30 y 60 segundos.

Sesion Redis activa detectada (HTTP 409)

El servidor tiene una sesion anterior que impide la re-autenticacion. Ejecuta este comando en la terminal del servidor para limpiar Redis:

docker compose exec redis redis-cli -a REDIS_PASSWORD FLUSHDB

Luego reinicia el ESP32 (boton EN o desconectar/reconectar USB).

Firmware IoT v2.0

Firmware modular para ESP32 que lee sensores de temperatura, humedad y ruido, controla actuadores (LED semaforo, IR) con logica local, y envia telemetria al servidor usando autenticacion criptografica de puzzle (HMAC-SHA256 + AES-256-CBC) con tokens JWT.

Protocolo de autenticacion
  1. Genera R2 (32 bytes aleatorios)
  2. Calcula P2 = HMAC-SHA256(device_key || server_key, R2)
  3. Cifra P2c = AES-256-CBC(P2, device_key, IV_random)
  4. Envia puzzle_response a POST /api/v1/auth/device/login
  5. Recibe JWT con expiracion de 24h (sesion unica en Redis)
  6. Usa JWT como Bearer token para POST /api/v1/device/reading

Limitaciones conocidas
  • El sensor de ruido se usa localmente para el LED semaforo pero no se envia a la API (no hay campo compatible).
  • Si el ESP32 se reinicia y la sesion Redis aun esta activa (TTL 24h), el dispositivo no puede autenticarse hasta que expire (error 409, "degraded mode"). Solucion: ejecutar docker compose exec redis redis-cli -a REDIS_PASSWORD FLUSHDB en el servidor y reiniciar el ESP32.
  • config.json almacena claves en texto plano. Para produccion, considerar flash encryption del ESP32.
  • Comunicacion HTTP sin TLS. El servidor tiene Nginx con TLS, pero urequests de MicroPython tiene soporte limitado.
  • Sin actualizaciones OTA, sin watchdog hardware, sin buffer local de datos offline.
